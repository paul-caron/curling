\doxysubsection{Detailed Description}
\begin{DoxyNote}{Note}
If you use Method\+::\+MIME (multipart POST), you must reset the \doxylink{classcurling_1_1Request}{Request} before switching to another method. Attempting to change it afterward throws logic\+\_\+error. 

Curling internally manages curl\+\_\+global\+\_\+init() and curl\+\_\+global\+\_\+cleanup() using std\+::once\+\_\+flag. You donâ€™t need to do this manually. 

Header keys in \doxylink{structcurling_1_1Response_ad36bcbd66306c7625242f31f4fe39c43}{Response\+::headers} are stored in lowercase to support case-\/insensitive lookup. 

By default, cookies are persisted in "{}cookies.\+txt"{}. Override this via set\+Cookie\+Path(). 

Calling enable\+Verbose(true) enables libcurl\textquotesingle{}s verbose output to stderr, which is useful for debugging. 
\end{DoxyNote}
